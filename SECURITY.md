🔐 Security Policy
📦 Supported Versions

The following versions of this project are currently supported with security updates:

Version	Supported
5.1.x	✅ Yes
5.0.x	❌ No
4.0.x	✅ Yes
< 4.0	❌ No

We strongly recommend upgrading to a supported version to ensure you receive the latest security patches and improvements.

🚨 Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

📩 How to Report
Send an email to: himanshusehgal320@gmail.com
Or create a private security advisory on GitHub (preferred for sensitive issues)
📝 What to Include

To help us resolve the issue quickly, include:

Description of the vulnerability
Steps to reproduce
Potential impact
Any proof-of-concept (screenshots, code, etc.)
⏱️ Response Timeline
Acknowledgment: Within 24–48 hours
Initial Assessment: Within 3–5 business days
Resolution / Patch: Depends on severity (usually within 7–14 days)
🔍 What to Expect
You will receive updates as we investigate the issue
If the vulnerability is accepted:
We will work on a fix and release a patch
You may be credited (if desired)
If declined:
We will provide a clear explanation
🔒 Responsible Disclosure

Please:

Do not publicly disclose the vulnerability before it is resolved
Avoid exploiting the issue beyond what is necessary for demonstration
